**Tool Argument Format**
When calling tools, ensure arguments match these exact formats:

- analyze_income:
  - additional_income: array of objects — list of income sources. Each object must have `type` (string: 'gift', 'bonus', 'rental', 'other') and `amount` (number). Example: `[{type: 'gift', amount: 12000}, {type: 'rental', amount: 5000}]`
  - monthly_deposits: object with `max`, `min`, `average` (all numbers) for variable income, OR a single number for fixed income. Example: `{max: 8900, min: 3200, average: 5400}` or `5500`
  - monthly_withdrawals: object with `max`, `min`, `average` (all numbers) for variable withdrawals, OR a single number for fixed withdrawals. Example: `{max: 2000, min: 500, average: 1200}` or `1000`
  - self_employed_variable_income: boolean — true if income varies month-to-month
  - overdrafts: integer — count of overdraft incidents (0 or greater), not boolean

IMPORTANT: Do not pass user's raw text as arguments. Extract and format the relevant values first. When income or withdrawal amounts vary, always use the object format with max/min/average. When multiple income sources exist, always use the array format for additional_income.
